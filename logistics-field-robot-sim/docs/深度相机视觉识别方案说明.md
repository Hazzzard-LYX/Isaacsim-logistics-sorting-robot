# 深度相机视觉识别方案说明

本文档说明如何在当前物流场地仿真项目中启用深度相机，并实现取货区豆子/液体颜色识别和放置区数字标签识别。

当前项目状态：

- 机器人已经有一个前向 RGB Camera prim。
- 当前视觉检测没有真正读取图像，而是直接读取 USD 自定义属性。
- 取货区颜色当前来自 `vision:liquidColor`。
- 放置区数字当前来自 `logistics:labelNumber`。

本文档描述后续如何替换为真实视觉流程：

```text
深度相机采集 RGB + Depth
        ↓
目标区域检测和分割
        ↓
颜色识别 / 数字识别
        ↓
输出识别结果
        ↓
驱动车顶指示灯或任务逻辑
```

## 1. 为什么使用深度相机

普通 RGB 相机只能提供颜色和纹理信息，无法直接获得目标距离和三维位置。深度相机可以同时提供：

1. RGB 图像：用于识别颜色、数字、标签形状。
2. Depth 图像：用于估计物体距离、分割前景目标、过滤背景干扰。
3. 相机内参：用于把像素点反投影到三维空间。

对于本项目，深度相机的主要作用有：

1. 确定箱体/标签是否在相机正前方。
2. 利用深度范围裁剪掉远处背景。
3. 提取箱子内部豆子或液体所在区域。
4. 提取放置区标签平面区域。
5. 提高颜色识别和数字识别的稳定性。

## 3. 取货区豆子/液体颜色识别原理

### 3.1 识别目标

取货区需要识别箱内物体颜色。

当前仿真中使用三种颜色：

```text
red
green
white
```

如果课程题目中实际是豆子，可以对应为：

```text
红色豆子 / 红色物料
绿色豆子 / 绿色物料
白色豆子 / 白色物料
```

识别输出仍然可以保持：

```text
red / green / white
```

### 3.2 基本流程

豆子颜色识别建议使用“深度辅助的颜色分割”。

完整流程：

```text
采集 RGB + Depth
        ↓
根据深度截取近距离目标区域
        ↓
定位透明箱内部区域
        ↓
提取候选颜色像素
        ↓
转换到 HSV 或 Lab 色彩空间
        ↓
统计主颜色
        ↓
输出 red / green / white
```

### 3.3 深度裁剪

机器人到达取货箱前时，箱子大致位于相机前方固定距离。

可以根据深度图筛选合理距离范围：

```text
min_depth <= depth <= max_depth
```

例如：

```text
0.20m <= depth <= 0.80m
```

这样可以去掉：

1. 太近的机器人自身结构。
2. 太远的地板、围栏、背景。
3. 非目标箱体区域。

伪代码：

```python
depth_mask = (depth > 0.20) & (depth < 0.80)
```

### 3.4 ROI 提取

由于机器人会先转向让相机正对箱子，目标通常位于图像中心区域。

可以先使用固定中心 ROI：

```text
图像宽度中间 50%
图像高度中间 50%
```

例如：

```python
h, w = rgb.shape[:2]
roi = rgb[int(0.25*h):int(0.75*h), int(0.25*w):int(0.75*w)]
roi_depth = depth[int(0.25*h):int(0.75*h), int(0.25*w):int(0.75*w)]
```

后续可升级为基于深度聚类的 ROI：

```text
用深度图找出相机前方最近的大块物体
        ↓
提取该物体的 2D bounding box
        ↓
在 bounding box 中识别颜色
```

### 3.5 颜色空间转换

直接在 RGB 空间判断颜色容易受光照影响。建议转换到 HSV 或 Lab。

HSV 的优点：

- Hue 表示色相，适合区分红/绿。
- Saturation 表示饱和度，适合区分彩色和白色。
- Value 表示亮度，可以过滤过暗区域。

颜色判断可采用规则：

```text
红色:
    Hue 接近 0° 或 180°
    Saturation 较高

绿色:
    Hue 接近 60° 到 90°
    Saturation 较高

白色:
    Saturation 较低
    Value 较高
```

OpenCV HSV 范围通常为：

```text
H: 0 ~ 179
S: 0 ~ 255
V: 0 ~ 255
```

可设置初始阈值：

```text
red:
  H: 0~10 或 170~179
  S: > 80
  V: > 80

green:
  H: 35~90
  S: > 60
  V: > 60

white:
  S: < 45
  V: > 140
```

### 3.6 主颜色判定

对 ROI 中满足深度条件的像素进行颜色分类，统计各类别像素数量：

```python
red_count = count(red_mask)
green_count = count(green_mask)
white_count = count(white_mask)
```

选择数量最多的类别：

```python
color = argmax({
    "red": red_count,
    "green": green_count,
    "white": white_count,
})
```

置信度可定义为：

```text
confidence = 最大类别像素数 / 三类有效像素总数
```

如果置信度太低，则返回 `None`，提示识别失败或需要重新对准。

### 3.7 豆子与箱体的区分

如果实际仿真中箱体是透明或半透明的，箱壁可能影响颜色判断。可使用深度和区域约束减少干扰：

1. 只取箱体内部较低区域。
2. 忽略边缘高反光区域。
3. 使用形态学开运算去除小噪声。
4. 只统计面积较大的连通区域。

推荐 ROI 进一步缩小为：

```text
图像中心区域中下部
```

因为豆子/液体通常位于箱体底部。

示意：

```text
+----------------------+
|      箱体上沿         |
|                      |
|    只取中下部 ROI     |
|      豆子/液体区域    |
+----------------------+
```

## 4. 放置区数字标签识别原理

### 4.1 识别目标

放置区需要识别箱体侧面的数字标签。

当前仿真中数字标签为：

```text
1 / 2 / 3 / 4 / 5
```

标签特点：

1. 白色背景。
2. 红色数字。
3. 数字贴在箱体朝向场地中心的一侧。
4. 机器人到达目标点后，相机正对标签。

### 4.2 基本流程

数字标签识别建议使用“深度辅助的标签定位 + 图像二值化 + 数字分类”。

完整流程：

```text
采集 RGB + Depth
        ↓
利用深度裁剪目标距离
        ↓
检测白色标签区域
        ↓
透视矫正或裁剪标签 ROI
        ↓
提取红色数字
        ↓
数字分类
        ↓
输出 1~5
```

### 4.3 深度辅助标签定位

机器人正对放置箱时，标签位于相机前方较近位置。先用深度图去掉背景：

```python
depth_mask = (depth > 0.20) & (depth < 0.90)
```

然后在 RGB 图像中检测白色标签背景：

```text
白色标签:
    Saturation 较低
    Value 较高
```

HSV 阈值示例：

```text
S < 60
V > 160
```

将白色 mask 与深度 mask 相与：

```python
label_candidate_mask = white_mask & depth_mask
```

再进行连通域分析，选择面积合适、形状接近方形的区域作为标签。

### 4.4 标签 ROI 裁剪

找到标签区域后，可以计算 bounding box：

```text
x_min, y_min, x_max, y_max
```

然后裁剪标签 ROI：

```python
label_roi = rgb[y_min:y_max, x_min:x_max]
```

为了减少边缘干扰，可以向内缩小 5%~10%：

```text
只保留标签中心区域
```

如果标签存在透视倾斜，可进一步使用轮廓四点做透视矫正：

```text
检测标签四边形轮廓
        ↓
排序四个角点
        ↓
warpPerspective 到固定大小
```

当前仿真中机器人会正对标签，初期可以先不做透视矫正，只用 bounding box 裁剪。

### 4.5 红色数字提取

标签背景是白色，数字是红色。因此数字提取可以用 HSV 红色阈值：

```text
red:
  H: 0~10 或 170~179
  S: > 80
  V: > 80
```

得到红色数字 mask：

```python
digit_mask = red_mask(label_roi)
```

然后进行形态学处理：

1. 开运算去掉小噪点。
2. 闭运算连接断裂笔画。
3. 取最大连通区域或所有红色笔画区域。

### 4.6 数字分类方案一：七段数码管规则识别

当前仿真中的数字由七段数码管样式的红色长方体组成。因此最适合使用七段规则识别，不需要训练神经网络。

七段编号：

```text
   a
 f   b
   g
 e   c
   d
```

数字对应关系：

```text
1: b, c
2: a, b, g, e, d
3: a, b, g, c, d
4: f, g, b, c
5: a, f, g, c, d
```

识别步骤：

1. 将标签 ROI 归一化到固定大小，例如 100 x 100。
2. 将数字 mask 划分为七个检测区域。
3. 统计每个检测区域中的红色像素占比。
4. 如果某区域红色像素比例超过阈值，则认为该段点亮。
5. 根据点亮段组合匹配数字。

例如：

```python
segments_on = {
    "a": ratio(region_a) > threshold,
    "b": ratio(region_b) > threshold,
    "c": ratio(region_c) > threshold,
    "d": ratio(region_d) > threshold,
    "e": ratio(region_e) > threshold,
    "f": ratio(region_f) > threshold,
    "g": ratio(region_g) > threshold,
}
```

然后查表：

```python
SEGMENT_TO_DIGIT = {
    ("b", "c"): 1,
    ("a", "b", "g", "e", "d"): 2,
    ("a", "b", "g", "c", "d"): 3,
    ("f", "g", "b", "c"): 4,
    ("a", "f", "g", "c", "d"): 5,
}
```

该方法优点：

1. 不需要训练数据。
2. 可解释性强。
3. 与当前仿真标签生成方式完全匹配。
4. 对 1~5 的少量数字识别足够稳定。

### 4.7 数字分类方案二：模板匹配

如果数字不是严格七段样式，也可以使用模板匹配。

流程：

1. 为 1~5 准备标准二值模板。
2. 将检测到的数字 mask resize 到统一大小。
3. 和每个模板计算相似度。
4. 选择相似度最高的数字。

相似度可以使用：

```text
交并比 IoU
归一化相关系数
像素差平方和
```

这种方法也不需要训练，但要求标签字体、大小、旋转角度比较稳定。

### 4.8 数字分类方案三：OCR 或小型 CNN

如果后续数字标签样式变化较大，可以使用 OCR 或小型 CNN。

可选方案：

1. OCR：例如 Tesseract、EasyOCR。
2. 小型 CNN：训练 1~5 的分类器。
3. 目标检测模型：直接检测标签和数字。

但对于当前课程设计，优先推荐七段规则识别，因为当前数字就是七段几何体生成的，规则方法更简单、可靠、可解释。

## 5. 深度信息如何辅助识别

深度图不仅用于裁剪背景，还可以用于判断目标是否对准。

### 5.1 判断目标距离

对检测到的 ROI，计算中位深度：

```python
target_depth = median(depth[roi_mask])
```

如果目标距离超出范围：

```text
target_depth < 0.20m 或 target_depth > 0.90m
```

则认为当前没有正确看到目标。

### 5.2 判断目标是否居中

检测到标签或豆子区域后，计算其像素中心：

```text
center_x = (x_min + x_max) / 2
center_y = (y_min + y_max) / 2
```

与图像中心比较：

```text
abs(center_x - image_center_x) < threshold
```

如果偏差较大，可以让机器人微调朝向。

### 5.3 三维位置估计

已知相机内参：

```text
fx, fy, cx, cy
```

像素 `(u, v)` 和深度 `z` 可以反投影到相机坐标系：

```text
X = (u - cx) * z / fx
Y = (v - cy) * z / fy
Z = z
```

这可以用于估计标签或箱体相对于相机的位置，后续支持更精确的停车和对准。

## 9. 识别失败处理

实际视觉识别中可能出现以下失败情况：

1. 相机没有正对箱子。
2. 目标距离过远或过近。
3. 透明箱体反光导致颜色偏差。
4. 标签被箱体边缘遮挡。
5. 图像曝光过亮或过暗。
6. 数字红色区域断裂。

建议失败处理策略：

```text
如果 confidence < 阈值:
    等待下一帧再识别
    如果连续失败:
        机器人小角度左右扫描
    如果仍失败:
        返回 unknown 并灭灯
```
